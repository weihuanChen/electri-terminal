Task:
Redesign the page architecture of this industrial B2B website.

This is not a SaaS site and not a generic brand landing page.
It is a product-driven industrial catalog website focused on:
- category discovery
- technical trust
- specification readability
- downloads
- RFQ conversion

Core categories:
- Terminal Blocks
- Cable Glands
- Electrical Enclosures
- DIN Rail Accessories

Main problems with the current design:
- too generic
- too marketing-style
- weak category entry
- weak technical trust
- weak industrial product structure

Please redesign these page templates:
1. Homepage
2. Category page
3. Product detail page

Design principles:
- structured
- technical
- trustworthy
- modular
- reusable
- industrial B2B style

Please provide:
1. Overall page architecture strategy
2. Homepage wireframe
3. Category page wireframe
4. Product detail page wireframe
5. Shared reusable component system
6. Recommendations for typography, spacing, card density, and CTA hierarchy
7. A suggested Next.js folder/component structure

Additional design constraints:
- Do not use oversized abstract gradients as the main visual language
- Do not make the hero feel like a software company landing page
- Do not prioritize branding over product discovery
- Do not hide technical information too deep
- Use product categories and product series as the main navigation logic
- Treat downloads, certifications, and inquiry actions as first-class UI elements
