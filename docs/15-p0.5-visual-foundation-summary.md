# P0.5 视觉基础改造总结（MVP）

## 1. 阶段定位

`P0.5` 是 `P0` 与 `P1` 之间的视觉基础阶段，目的不是重做页面信息架构，而是先统一视觉语言，避免后续首页、分类页、产品页重复返工。

本阶段直接服务于以下目标：
- 统一深色工业风基底（不是纯黑）
- 保证内容承载区可读性和 SEO 友好
- 让 CTA 层级清晰、克制、有秩序
- 将视觉规范沉淀为可复用 token 和组件规则

---

## 2. 输入与约束

来自已确认的前提：
- 站点定位：小垂直权威站，主 `ring terminals`，副 `cable gland`
- 风格关键词：`Clean / Industrial / Minimal / Authentic / Structured`
- 主色方向：黑色系 + 高压橙 + 蓝色点缀
- 颜色预算：`70% 深色中性` + `20% 浅色内容区` + `10% 强调色`

本阶段不做：
- 首页信息架构重排（属于 `P1`）
- 分类内容扩充（属于 `P2`）
- 10-20 产品页深度补齐（属于 `P3`）

---

## 3. 视觉系统总纲（落地口径）

## 3.1 色彩策略（70/20/10）

- 深色中性（70%）
  - `#12161B` / `#1A1F26` / `#232A33`
  - 用于：Header、Hero、分隔带、Footer、深色信息模块
- 浅色内容区（20%）
  - `#FFFFFF` / `#EEF1F4` / `#D5DAE1`
  - 用于：正文承载区、规格表、FAQ、长文阅读块
- 强调色（10%）
  - 橙：`#F97316`（主 CTA）
  - 蓝：`#2F6EA8`（导航 active、次级强调）
  - 建议分配：橙 7%，蓝 3%

关键约束：
- 禁用大面积 `#000000`
- 同一视觉焦点区域避免橙蓝双主 CTA 竞争

## 3.2 组件语义

- 主按钮：橙色填充，承担“询盘/提交/主要转化”
- 次按钮：深色描边或蓝色描边，承担“查看详情/辅助动作”
- 卡片：弱阴影 + 清晰边框，不做营销化大阴影
- 表格：优先扫描效率（行分隔、对齐、单位一致）
- 链接：蓝色用于技术跳转和 active 状态

---

## 4. 代码层改造范围（P0.5）

## 4.1 全局层

目标文件：
- `app/globals.css`

改造内容：
- 建立统一 token（neutral / surface / accent / text / border / focus）
- 统一 `.btn` / `.card` / `.section` / 表单输入基础样式
- 调整暗色区与亮色区的默认对比，保证可读性

## 4.2 布局层

目标组件：
- `components/layout/HeaderClient.tsx`
- `components/layout/Footer.tsx`

改造内容：
- Header 固定深色工业底，active 使用蓝色提示
- 顶部主 CTA 使用橙色
- Footer 信息层级和文字对比度统一

## 4.3 共享组件层

目标组件：
- `components/shared/*`（按钮、卡片、表格、FAQ、下载卡、表单）

改造内容：
- 控件半径、边框、交互态统一
- 文档支持类模块视觉权重统一（不做“可立即下载”误导）
- 表格和参数区使用浅色高可读承载块

## 4.4 模板层

目标页面：
- 首页模板
- 分类页模板
- 产品页模板
- Blog 列表/详情模板

改造内容：
- 区块背景切换遵循 70/20/10
- CTA 位置和视觉主次一致
- 统一图片裁切、遮罩强度和文字对比

---

## 5. 执行顺序与检查清单

## 5.1 执行顺序

1. `globals.css` token 与基础控件
2. Header / Footer
3. 共享组件（按钮、卡片、表格、表单）
4. 首页模板统一
5. 分类页/产品页/Blog 模板统一

## 5.2 验收清单（P0.5 Gate）

- 视觉一致性
  - 首页/分类/产品/Blog 的颜色和控件语言一致
- 色彩配比
  - 接近 70/20/10，且无纯黑大底
- 转化清晰性
  - 首屏主 CTA 明确，且主次 CTA 不冲突
- 可读性
  - 规格表、FAQ、长文块在移动端和桌面端均可读
- 工业感
  - 去除 SaaS 风/营销大渐变风，保留结构化信息表达

---

## 6. 与后续阶段的接口

- 对 `P1`：提供稳定视觉基座，首页仅改信息结构和内容优先级，不再反复改样式系统
- 对 `P2`：分类页复用同一组件体系，新增 FAQ/列表块无需单独定制视觉
- 对 `P3`：产品页参数区、应用说明区、询盘区沿用同一 token 和组件规则
